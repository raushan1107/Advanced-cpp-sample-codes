üï∞Ô∏è The Story Begins ‚Äî ‚ÄúThe Age of C‚Äù

Let‚Äôs rewind to the 1970s. Computers were massive, slow, and could barely do what our phones do now. Dennis Ritchie created C, a simple and powerful language that talks close to the hardware ‚Äî ideal for building operating systems like UNIX.

C gave total control: manual memory management, passing the same data between functions repeatedly, and thinking more like the machine than a human.

‚öôÔ∏è Example: The Bank in the C Era

Here is a compact C implementation that models an account and operations on it.

#include <stdio.h>
#include <string.h>

struct Account {
    int accNo;
    char name[50];
    float balance;
};

void deposit(struct Account* acc, float amt) {
    acc->balance += amt;
}

void withdraw(struct Account* acc, float amt) {
    if (amt <= acc->balance)
        acc->balance -= amt;
    else
        printf("Insufficient balance!\n");
}

int main() {
    struct Account a1 = {101, "Ritu", 1000.0};
    deposit(&a1, 200);
    withdraw(&a1, 1500);
    printf("Balance for %s: %.2f\n", a1.name, a1.balance);
    return 0;
}

This works ‚Äî but notice the repetitive passing of the Account structure and the lack of built-in protection for data.

üöÄ The Problem that Started the Revolution

As software grew, developers asked: ‚ÄúWhy can‚Äôt code model the real world?‚Äù If a bank account is an entity, it should know how to deposit or withdraw itself ‚Äî not be manipulated by external functions.

Enter Bjarne Stroustrup in the early 1980s: C with Classes. Instead of deposit(account), we say account.deposit() ‚Äî logic lives inside the object.

üß© Example: The Bank in C++

Rewriting the earlier program using classes, constructors, and methods.

#include <iostream>
#include <string>
using namespace std;

class Account {
private:
    int accNo;
    string name;
    float balance;

public:
    // Constructor
    Account(int a, string n, float b) {
        accNo = a;
        name = n;
        balance = b;
    }

    void deposit(float amt) { balance += amt; }

    void withdraw(float amt) {
        if (amt <= balance)
            balance -= amt;
        else
            cout << "Insufficient balance!" << endl;
    }

    void showBalance() {
        cout << "Account: " << accNo << " | Holder: " << name
             << " | Balance: " << balance << endl;
    }
};

int main() {
    Account ritu(101, "Ritu", 1000.0);
    ritu.deposit(500);
    ritu.withdraw(200);
    ritu.showBalance();
}

Now behavior and data live together. Objects simplify reasoning, maintenance, and extension.

üß† Extras: Constructors, Static Members & Friends

Constructors / Destructors automatically handle setup and cleanup.

class Demo {
public:
    Demo() { cout << "Object created!\n"; }
    ~Demo() { cout << "Object destroyed!\n"; }
};

int main() {
    Demo d1;
} // destructor called here

Static members belong to the class itself (shared state like total accounts).

Friend functions allow trusted external access when necessary ‚Äî use sparingly.

üß≠ Mini Case Study ‚Äî Branch System

#include <iostream>
#include <string>
using namespace std;

class Account {
    int accNo;
    string holder;
    double balance;
    static int totalAccounts;

public:
    Account(string name, double amount) {
        holder = name;
        balance = amount;
        accNo = ++totalAccounts;
        cout << "Account Created for " << holder << " | Account No: " << accNo << endl;
    }

    void deposit(double amt) { balance += amt; }

    void withdraw(double amt) {
        if (amt <= balance) balance -= amt;
        else cout << "Insufficient funds!\n";
    }

    void showDetails() {
        cout << "Account No: " << accNo << " | Holder: " << holder
             << " | Balance: " << balance << endl;
    }

    ~Account() { cout << "Account Closed: " << holder << endl; }
};

int Account::totalAccounts = 0;

int main() {
    Account a1("Ritu", 1000);
    Account a2("Raushan", 2000);

    a1.deposit(500);
    a2.withdraw(700);

    a1.showDetails();
    a2.showDetails();
}
